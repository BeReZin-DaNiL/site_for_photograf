# Руководство по проекту "World Photo"

Это документ содержит подробную инструкцию по использованию и разработке веб-платформы для поиска фотографов "World Photo".

---

## Часть 1. Руководство Пользователя

### 1. Введение
**World Photo** — это платформа, соединяющая клиентов и профессиональных фотографов. 
- **Клиенты** могут искать специалистов, просматривать портфолио и оставлять заявки на съемку.
- **Фотографы** могут размещать свое портфолио, указывать цены и специализацию, а также получать заказы.

### 2. Регистрация и Вход
1.  Нажмите кнопку **"Регистрация"** в верхнем меню.
2.  Заполните форму: Имя пользователя, Email, Пароль.
3.  **Важно:** Выберите роль:
    *   **Я ищу фотографа** (Клиент) — для поиска и заказа услуг.
    *   **Я фотограф** (Фотограф) — для создания профиля специалиста.
4.  После регистрации вы будете автоматически перенаправлены в Личный кабинет.

### 3. Личный кабинет
Доступен по клику на аватарку в верхнем углу -> **"Профиль"**.

#### Для Всех:
*   **Редактирование профиля:** Можно изменить Имя, Фамилию, Email и **Фото профиля**.
*   **Фото профиля:**
    *   Нажмите "Изменить", чтобы загрузить новое фото.
    *   Фото можно удалить только путем замены на новое (кнопка удаления скрыта для предотвращения случайных действий).
*   **Настройки:**
    *   Управление уведомлениями.
    *   Смена пароля.
    *   **Удаление аккаунта:** Внизу раздела настроек находится кнопка "Удалить аккаунт". Это действие необратимо и полностью удаляет все данные пользователя.

#### Для Фотографов:
*   **Портфолио:** В личном кабинете есть блок "Ваше портфолио", где можно загружать примеры работ.
*   **Информация:** Заполните поля "О себе", "Специализация", "Цена", "Город", чтобы клиенты могли вас найти. Если поле "О себе" пустое, соответствующий блок в публичном профиле будет скрыт.

### 4. Поиск и Бронирование (Для Клиентов)
1.  Перейдите на страницу **"Специалисты"**.
2.  Используйте фильтры (Город, Цена, Специализация), чтобы найти подходящего фотографа.
3.  Перейдите в профиль фотографа.
4.  **Оставить заявку:**
    *   Справа (или внизу на мобильных) находится форма заявки.
    *   **Телефон:** Введите номер. Работает строгая маска ввода: `+ 7 999 999 99 99`. Если номер введен не полностью, заявка не отправится.
    *   Нажмите "Отправить заявку".

### 5. Управление Заявками (Booking)
Все заявки находятся в Личном кабинете во вкладке **"Заявки"** (или "Мои заказы").

#### Статусы заявок:
*   `Новая` — заявка отправлена, но еще не просмотрена.
*   `В работе` — фотограф принял заказ.
*   `Выполнена` — заказ завершен.
*   `Отменена` — заказ отменен одной из сторон.

#### Отмена и Удаление:
1.  **Если Фотограф нажимает "Отменить заказ":**
    *   Заявка **не удаляется**.
    *   Статус меняется на **"Отменена"**.
    *   Клиент видит этот статус (уведомление об отказе).
2.  **Если заявка уже имеет статус "Отменена":**
    *   Появляется кнопка **"Удалить"**.
    *   При нажатии заявка полностью стирается из истории.
3.  **Клиент** также может отменить/удалить свою заявку.

## Часть 2. Руководство Разработчика

### 1. Технический Стек
*   **Backend:** Python 3.x, Django 5.x
*   **Database:** SQLite (по умолчанию)
*   **Frontend:** HTML5, CSS3 (Custom styles + Flexbox/Grid), JavaScript (Vanilla)
*   **Template Engine:** Django Templates (DTL)

### 2. Структура Проекта
```
сайт для фотографов/
├── manage.py            # Утилита управления Django
├── db.sqlite3           # База данных
├── myproject/           # Конфигурация проекта (settings.py, urls.py)
├── users/               # Основное приложение
│   ├── models.py        # Модели (User, Profile, Booking, Photo)
│   ├── views.py         # Логика контроллеров
│   ├── forms.py         # Формы Django
│   ├── templates/       # HTML шаблоны
│   └── ...
├── static/              # Статические файлы (CSS, JS, Images)
└── media/               # Загружаемый контент (аватарки, портфолио)
```

### 3. Установка и Запуск
1.  **Установите Python**.
2.  **Создайте виртуальное окружение** (рекомендуется):
    ```bash
    python -m venv venv
    venv\Scripts\activate  # Windows
    # source venv/bin/activate  # Mac/Linux
    ```
3.  **Установите зависимости**:
    ```bash
    pip install django pillow
    ```
4.  **Примените миграции**:
    ```bash
    python manage.py migrate
    ```
5.  **Запустите сервер**:
    ```bash
    python manage.py runserver
    ```
    Сайт будет доступен по адресу: `http://127.0.0.1:8000/`

### 4. Ключевые Особенности Реализации

#### 4.1. Валидация Телефона
Реализована на двух уровнях:
1.  **Frontend (JS)**: В `photographer_detail.html` скрипт перехватывает ввод в поле `#phone-input`, накладывая маску `+ 7 XXX XXX XX XX` и блокируя удаление префикса.
2.  **Backend (Python)**: В `forms.py` метод `clean_contact_phone` проверяет соответствие регулярному выражению.

#### 4.2. Логика Отмены (Views)
В `views.py` (функция `dashboard`):
*   Обработка POST-запроса `cancel_booking`.
*   Если инициатор **Фотограф** и статус != `cancelled`: устанавливается статус `cancelled` (soft cancel).
*   В остальных случаях (или если уже отменена): выполняется `booking.delete()` (hard delete).

#### 4.3. Стилизация
*   Все стили находятся в `static/styles.css`.
*   Используются CSS Variables (`:root`) для легкой смены цветовой темы.
*   Адаптивная верстка реализована через `@media` запросы.

#### 4.4. (Уведомления)
*   Используется Django Messages Framework (`messages.success`, `messages.error`).
*   В `base.html` JS-скрипт проверяет наличие сообщений и показывает кастомный тост-уведомление (зеленый для успеха).
