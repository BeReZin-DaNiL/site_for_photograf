# Инструкция по запуску и публикации сайта

Этот документ поможет вам запустить проект на компьютере и загрузить его на GitHub.

---

## Часть 1: Запуск сайта (для проверки или разработки)

### 1. Подготовка
Убедитесь, что у вас установлен **Python** (версии 3.10 или выше).
Проверить версию можно командой:
```bash
python --version
```

### 2. Установка зависимостей
Откройте терминал (командную строку) в папке с проектом и выполните следующие шаги:

**Шаг 1. Создайте виртуальное окружение (рекомендуется):**
```bash
python -m venv venv
```

**Шаг 2. Активируйте виртуальное окружение:**
*   **Windows:**
    ```bash
    venv\Scripts\activate
    ```
*   **Mac/Linux:**
    ```bash
    source venv/bin/activate
    ```
*(После активации в начале строки терминала появится надпись `(venv)`)*

**Шаг 3. Установите необходимые библиотеки:**
```bash
pip install -r requirements.txt
```

### 3. Подготовка базы данных
Если вы запускаете проект впервые или база данных удалена:
```bash
python manage.py migrate
```

*(Опционально) Создайте администратора для доступа в админ-панель:*
```bash
python manage.py createsuperuser
```

### 4. Запуск сервера
Запустите веб-сервер командой:
```bash
python manage.py runserver
```

Теперь сайт доступен по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

## Часть 2: Загрузка сайта на GitHub

### 1. Регистрация и создание репозитория
1.  Зарегистрируйтесь на [github.com](https://github.com/).
2.  Нажмите **"+"** в правом верхнем углу -> **"New repository"**.
3.  Придумайте название (например, `world-photo-site`).
4.  Выберите **Public** (публчный) или **Private** (частный).
5.  **Не ставьте** галочки "Add a README file" или ".gitignore" (мы добавим их сами).
6.  Нажмите **"Create repository"**.

### 2. Загрузка кода (через терминал)
Убедитесь, что вы находитесь в корневой папке проекта (где лежит файл `manage.py`).

**Шаг 1. Инициализация Git:**
```bash
git init
```
*(Эта команда создает скрытую папку .git)*

**Шаг 2. Добавление файлов:**
Мы заранее создали файл `.gitignore`, чтобы не загружать лишний мусор (виртуальное окружение, временные файлы).
```bash
git add .
```

**Шаг 3. Сохранение изменений (коммит):**
```bash
git commit -m "Первая версия сайта: запуск проекта"
```

**Шаг 4. Выбор ветки (обычно main):**
```bash
git branch -M main
```

**Шаг 5. Привязка к GitHub:**
Скопируйте ссылку на ваш репозиторий с сайта GitHub (она выглядит как `https://github.com/ВашЛогин/world-photo-site.git`) и выполните команду:
```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
```
*(Замените ссылку на свою!)*

**Шаг 6. Отправка файлов:**
```bash
git push -u origin main
```

### 3. Готово!
Обновите страницу вашего репозитория на GitHub — вы увидите все файлы вашего проекта.

---

### Полезные команды

*   **Посмотреть статус файлов:** `git status`
*   **Добавить новые изменения:**
    ```bash
    git add .
    git commit -m "Описание того, что изменили"
    git push
    ```
